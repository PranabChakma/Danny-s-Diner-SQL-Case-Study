# Danny's Diner SQL Case Study

# Overview
Danny owns a small restaurant and wants to undersatnd his customers' spending and dining patterns.
Using SQL this project explores insights such as how much customers spend, their favourite dishes, and how membership impacts behavior.

This project is based on the case study **"Danny's Dinner"** from the 8 week SQL challenge

# Dataset
This project uses three tables:
**Sales**: Records of customer orders (Customer_id, Order_date, Product_id)
**Menu**: Menu items and their prices (Product_id, Product_name, Prices)
**Members**: Membership join date (Customer_id, Join_date)

# Case Study Questions
1. Total amount each customer spent at the restaurant
2. Number of days each customer visited
3. First item purchased by each customer
4. Most purchased menu item overall
5. Most popular item for each customer
6. First item purchased after becoming a member
7. Item purchased just before becoming a member
8. Total items and amount spent before membership
9. Points earned (10 points per $1, sushi = 2x points)
10. Points for customers A & B at the end of january (2x multiplier for first week after joining)
